## 0. 前置准备

### 0.1 环境检查清单

| 组件 | 端口 | 检查方式 |
|---|---|---|
| MySQL | 3306 | 确认已执行 `sql/schema-*.sql` 所有建表脚本 |
| Nacos | 8848 | 浏览器打开 `http://localhost:8848/nacos`，看服务列表 |
| campus-user-auth-service | 9010 | Nacos 控制台能看到该服务 |
| campus-market-service | 9020 | Nacos 控制台能看到该服务 |
| campus-task-service | 9030 | Nacos 控制台能看到该服务 |
| campus-activity-service | 9040 | Nacos 控制台能看到该服务 |
| campus-notification-service | 9050 | Nacos 控制台能看到该服务 |
| campus-gateway | 9000 | Nacos 控制台能看到该服务 |

### 0.2 启动后端服务（顺序不能乱）

```
1. MySQL
2. Nacos
3. campus-user-auth-service      ← 必须先启，其他服务 Feign 依赖它
4. campus-market-service
5. campus-task-service
6. campus-activity-service
7. campus-notification-service
8. campus-gateway                 ← 最后启，前端所有请求走它
```

### 0.3 启动前端

```bash
cd frontend
npm install      # 仅首次
npm run dev      # 启动后打开 http://localhost:5173
```

## 9. 常见问题排查

| 现象 | 可能原因 | 排查方法 |
|---|---|---|
| 注册提示失败 | user-auth-service 没启动，或数据库密码不对 | 检查 Nacos 上该服务是否在线；查看后端控制台报错 |
| 登录提示失败 | 账号密码错误，或 Gateway 没启动 | 确认用了正确的学号和密码；检查 Nacos 上 Gateway 是否在线 |
| 首页统计数据全为 0 | 对应业务服务没启动 | 检查 Nacos 服务列表，确认 market/task/activity 服务都在线 |
| 商品列表为空 | market-service 没启动或数据库问题 | 检查 Nacos + 后端控制台 |
| 发布商品时报错、无响应 | Gateway 没转发到正确的服务 | 确认 Gateway 在最后启动，Nacos 上所有服务均在线 |
| 接单后状态没更新 | 前端做了乐观更新（本地直接改状态），刷新页面看后端数据是否一致 | 刷新页面，如果状态回退说明后端接口未成功 |
| 商品列表看不到别人的商品 | status 过滤问题，列表默认只显示 ON_SHELF | 让发布者确认商品是"上架中"状态 |
| 创建社团找不到入口 | 前端部分页面入口可能在具体业务页面 | 直接在地址栏输入 `/clubs` 尝试访问社团列表页 |
| npm 报错 | Node.js 未安装或不在 PATH 中 | 运行 `node --version` 检查；如未安装去 nodejs.org 下载 LTS 版 |
| 页面空白白屏 | 前端控制台有 JS 报错 | 按 F12 → Console，查看报错信息 |
